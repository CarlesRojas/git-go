# Git Go Extension - Development Instructions

## System Prompt / Instructions

### CSS/Styling Guidelines

**Layout & Spacing Rules:**

- **Avoid nested divs** - Never use divs inside divs if they have no other child elements
- **Use flexbox + gap** - Always prioritize `flex` and `gap` for spacing instead of margins
- **No child margins** - Never use margins on child elements for spacing
- **No Tailwind space utilities** - Do not use `space-x-*`, `space-y-*`, or similar spacing methods

**Code Guidelines:**

- **No comments** - Never include comments in the code. Code should be self-explanatory through clear naming and structure
- **Empty lines between HTML siblings** - Always add an empty line between sibling HTML elements for better readability
- **Use opacity for text colors** - Never use explicit Tailwind text colors like `text-gray-400`. Use `opacity-*` classes instead to work with VS Code theme variables

**Preferred Pattern:**

```jsx
// ✅ Good - flex with gap
<div className="flex gap-3">
  <span>Item 1</span>
  <span>Item 2</span>
</div>

// ❌ Bad - nested divs with margins
<div>
  <div className="mr-4">Item 1</div>
  <div>Item 2</div>
</div>
```

**HTML Formatting:**

```jsx
// ✅ Good - empty lines between siblings
<div>
  <header>Header content</header>

  <main>Main content</main>

  <footer>Footer content</footer>
</div>

// ❌ Bad - no spacing between siblings
<div>
  <header>Header content</header>
  <main>Main content</main>
  <footer>Footer content</footer>
</div>
```
